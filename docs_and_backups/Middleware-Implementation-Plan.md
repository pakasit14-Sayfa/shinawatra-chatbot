# โครงสร้างและแผนงาน Middleware (ตัวกลาง) เชื่อม LINE/Facebook กับ Dify

เนื่องจากเราเลือก **กรณีที่ 2** (มีระบบตัวกลาง) เพื่อความปลอดภัยสูงสุด และต้องการฟีเจอร์ **"แยกการตอบตามเพจ (Channel Isolation)"** และ **"แยกอารมณ์ตามเพจ (Dynamic Personality)"** นี่คือแผนภาพสถาปัตยกรรม (Architecture) ที่อัพเดทใหม่ครับ

## 🏗️ โครงสร้างการทำงาน (Architecture Flow)

1. **User:** พิมพ์ข้อความเข้ามา (ผ่าน เพจพยาบาล, เพจหลัก, หรือ LINE)
2. **Middleware (FastAPI):** รับข้อความและ **"ตรวจสอบแหล่งที่มา (Source Channel)"**
3. **Middleware (Context & Persona Injection):** นำแหล่งที่มาไปแนบกับคำถามก่อนส่งให้ Dify เพื่อตีกรอบให้บอทรู้ตัว ทั้งเรื่องเนื้อหาและอารมณ์
4. **Dify API:** ประมวลผลและตอบคำถามแบบไม่ข้ามเพจ และใช้อารมณ์ที่ถูกต้อง
5. **Middleware (Interceptor):** 
   - รับคำตอบจาก Dify
   - **นำคำตอบเข้า `validate_contact.py`** เพื่อเช็ค Contact Leak (เบอร์หลุด)
6. **Decision Engine:**
   - ✅ **Safe:** ส่งคำตอบกลับไปหาผู้ใช้
   - 🚨 **Leak:** สกัดกั้นคำตอบ เปลี่ยนข้อความ และยิง Alert ลง Slack

## 🔄 ระบบสลับการทำงานระหว่างบอทกับแอดมิน (Bot/Admin Handoff Mode)
เพื่อแก้ปัญหา "บอทแย่งแอดมินตอบ" 
- **Admin Mode (Takeover):** เมื่อแอดมินพิมพ์คำสั่ง `/stop` หรือบอทเจอคำว่า "ติดต่อแอดมิน" ตัวกลางจะตัดการทำงานของบอททันที ปล่อยให้แอดมินคุยต่อ

## 🧱 ระบบตีกรอบเพจและอารมณ์ (Channel Isolation & Dynamic Personality)

| แหล่งที่มา (Source) | คำสั่งลับที่ Middleware จะกระซิบให้ Dify (System Prompt Injection) | บุคลิกและผลลัพธ์ที่จะออกมา |
| :--- | :--- | :--- |
| **Facebook เพจพยาบาล** | "[System: สวมบทบาทเป็น 'อาจารย์' ห้ามตอบเรื่อง ป.โท]" | บอทจะน่ารัก น่าเชื่อถือ ดูแลใส่ใจแบบอาจารย์ และปฏิเสธการตอบเรื่อง ป.โท |
| **Facebook เพจหลัก** | "[System: สวมบทบาท 'แอดมินทางการ' ห้ามตอบเรื่องพยาบาล]" | บอทจะเป็นผู้ใหญ่ น่าเชื่อถือ ดูเป็นมืออาชีพสำหรับวัยทำงาน |
| **LINE OA (รวม)** | "[System: ตอบแบบ 'Hybrid' สุภาพและเป็นมิตร]" | บอทจะถามและนำทางผู้ใช้เพื่อแยกกลุ่มเป้าหมาย |
