# 🐣 WEEK 3: Canary Deploy (Soft Launch)

**Status:** เริ่มต้นทดสอบกับผู้ใช้งานจริงกลุ่มเล็ก
**Goal:** เปิดให้ใช้งานจริงในวงจำกัด ควบคุมความเสี่ยง และหา Edge Cases

---

## 🎯 **Week 3 Overview (7 วัน)**

```
Day 15-16: Soft Launch Setup & Execution
Day 17-18: Active Monitoring (เฝ้าระวังอย่างใกล้ชิด)
Day 19-20: Prompt Tuning & Hotfixes (แก้บั๊กรายวัน)
Day 21:    Go / No-Go Decision (ประเมินก่อนขึ้นระบบจริง)
```

---

## 📍 **ขั้นตอนที่ 1: Soft Launch Setup (Day 15-16)**

**ทำไร:** 
นำแชตบอทไปให้ผู้ใช้งานจริงกลุ่มเล็กทดลองใช้ โดยยังไม่ประกาศอย่างเป็นทางการ (เช่น ไม่ยิงแอดโปรโมท)

**ขั้นตอน:**
1. **เตรียม Link / Widget:**
   - นำลิงก์แชร์ของ Dify ไปโพสต์ในกลุ่ม Facebook ปิด, ส่งใน LINE กลุ่มนักเรียน, หรือเอา Widget ไปติดที่หน้าเว็บเพจย่อย (Sub-page)
2. **จำกัดปริมาณ:**
   - พยายามให้มีคนใช้งานประมาณ 50-100 คนต่อวัน เพื่อไม่ให้โหลดหนักเกินไป
3. **เตรียมพร้อม Support:**
   - แจ้งทีมแอดมินให้ Standby เพื่อเข้าไป Take Over แชตได้ทันทีหากมีคนถามคำถามที่บอทตอบไม่ได้

---

## 📍 **ขั้นตอนที่ 2: Active Monitoring (Day 17-18)**

**ทำไร:**
เฝ้าดูการทำงานของบอทแบบ Real-time และเช็คแจ้งเตือนจากระบบ Support ที่เราสร้างไว้

**ขั้นตอน:**
1. **เปิด Slack ทิ้งไว้:**
   - ดูว่ามี Alert `🚨 CRITICAL: Contact Leak` เข้ามาหรือไม่
   - หากมี ให้รีบเข้าไปดู Log ใน Dify ว่าผู้ใช้พิมพ์อะไร บอทถึงหลุดเบอร์
2. **สุ่มตรวจ Log ใน Dify:**
   - เข้าไปที่เมนู **"Logs & Annotations"** ใน Dify
   - ไล่อ่านแชตของผู้ใช้จริง สังเกตว่า:
     - บอทถามซ้ำไหม? (Memory ทำงานปกติไหม)
     - บอทเข้าใจคำแสลง/คำย่อไหม? (เช่น กศน., ปวส.)
     - บอทประเมิน BMI ได้ถูกต้องไหมเมื่อผู้สมัคร RN ให้ข้อมูล?

---

## 📍 **ขั้นตอนที่ 3: Prompt Tuning & Hotfixes (Day 19-20)**

**ทำไร:**
ปรับจูนสมองของบอท (Prompt) ตามปัญหาที่พบจากการใช้งานจริง

**ขั้นตอน:**
1. **รวบรวมปัญหา:**
   - จดบันทึกคำถามที่บอทตอบผิด หรือตอบวกวน
2. **แก้ Prompt ทีละจุด:**
   - เข้าไปแก้ YAML ใน Dify (ตรงเมนู Prompt ของโหนดนั้นๆ)
   - *ตัวอย่าง:* ถ้าบอทไม่รู้จักวุฒิ "ม.ศ.5" ให้ไปเพิ่มในส่วนของ Synonym Rules
3. **ทดสอบก่อน Publish:**
   - ทุกครั้งที่แก้ ให้ลองพิมพ์คำถามที่เคยมีปัญหาลงในช่อง Preview ดูว่าแก้ได้จริงไหม
   - มั่นใจแล้วค่อยกด Publish เพื่ออัพเดทบอทบนเว็บ

---

## 📍 **ขั้นตอนที่ 4: Go / No-Go Decision (Day 21)**

**ทำไร:**
ประชุมทีมเพื่อตัดสินใจว่าจะเปิดให้คนทั้งประเทศใช้ (Go) หรือต้องเลื่อนเพื่อแก้ปัญหาใหญ่ก่อน (No-Go)

**Checklist สำหรับการผ่านด่าน:**
- [ ] มี Contact Leak เกิดขึ้นน้อยกว่า 1% หรือไม่มีเลย
- [ ] บอทสามารถเก็บข้อมูลผู้สมัคร (Lead Generation) ได้ครบถ้วนก่อนส่งให้แอดมิน
- [ ] ไม่เจอปัญหาบอทตอบวนลูปซ้ำซาก
- [ ] ผู้ใช้ส่วนใหญ่ (80%+) ได้รับคำตอบที่พึงพอใจโดยไม่ต้องรอแอดมิน

**สรุป:** หากติ๊กถูกครบทุกข้อ แปลว่าเราพร้อมสำหรับ **Week 4: Production (Go Live)** แล้ว! 🚀
