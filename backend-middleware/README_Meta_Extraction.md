# 📦 Meta Chat History Extraction Pipeline

ระบบสำหรับดึงประวัติการแชท (Chat History) จากเพจ Facebook หลายๆ เพจ พร้อมคัดกรองข้อมูลซ้ำ และแปลงให้อยู่ในรูปแบบ `.csv` อัตโนมัติ เพื่อนำไปใช้งานต่อกับระบบ **Knowledge Base (RAG)** ของ Dify

## 🚀 ฟีเจอร์หลัก
- ดึงข้อมูลจากหลายเพจพร้อมกัน (รองรับ 7 เพจผ่าน `.env`)
- ดึงชื่อเพจจริงจาก Facebook เพื่อใช้ตั้งชื่อไฟล์โดยอัตโนมัติ
- ดึงข้อมูลย้อนหลัง 180 วันล่าสุด (หรือตามที่ตั้งค่าในโค้ด)
- กรองคำถาม-คำตอบที่ไม่สมบูรณ์หรือสั้นเกินไปออก
- คัดกรองข้อมูลแชทที่ซ้ำกัน (Deduplication)
- แปลงเป็นไฟล์ CSV ทันที พร้อมสำหรับการทำ Knowledge Base Q&A

## 🛠️ โครงสร้างไฟล์
- `multiPagePipeline.js` : ตัวรันหลัก ที่จะวนลูปอ่านการตั้งค่าจาก `.env` เพื่อโหลดข้อมูลและบันทึกไฟล์
- `extractChatHistory.js` : ไฟล์ที่มีฟังก์ชันย่อยในการเชื่อมต่อกับ Meta Graph API
- `fullPipeline.js` / `cleanData.js` : ไฟล์สคริปต์เก่าที่ทำงานแยกกัน (เก็บไว้เป็นข้อมูลอ้างอิง)

## ⚙️ วิธีติดตั้งและการใช้งาน

### 1. ตั้งค่า `.env`
เพิ่ม `FB_PAGE_ID_X` และ `FB_PAGE_TOKEN_X` เข้าไปในไฟล์ `.env` โดย `X` คือลำดับของเพจตั้งแต่ 1 ถึง 7

ตัวอย่าง:
```env
FB_PAGE_ID_1=1234567890
FB_PAGE_TOKEN_1=EAAXXXXXXXXX...

FB_PAGE_ID_2=0987654321
FB_PAGE_TOKEN_2=EAAYYYYYYYYY...
```

### 2. รันสคริปต์
พิมพ์คำสั่งด้านล่างใน Terminal:
```bash
node multiPagePipeline.js
```

### 3. ผลลัพธ์
เมื่อสคริปต์ทำงานเสร็จ จะได้ไฟล์ `.csv` แยกตามชื่อเพจ เช่น:
- `training_data_Page1_ศูนย์รับสมัครนักศึกษา_มหาวิทยา_cleaned.csv`
- `training_data_Page2_มหาวิทยาลัยชินวัตร_cleaned.csv`

---

## 📚 การนำไปใช้กับ Dify
1. ไปที่เมนู **Knowledge** ใน Dify -> **Create Knowledge**
2. อัปโหลดไฟล์ CSV ที่ได้จากระบบ
3. เลือกรูปแบบ (Format) เป็นแบบ **Q&A** (สำคัญมาก)
4. เมื่อประมวลผลเสร็จแล้ว สามารถนำ Knowledge นี้ไปผูกกับโหนด LLM ของบอทแต่ละสาขาได้เลย!
