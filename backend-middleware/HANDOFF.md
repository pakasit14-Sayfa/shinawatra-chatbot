# Handoff — สถานะระบบ ณ วันนี้

อ่านไฟล์นี้ก่อนเริ่มงานต่อ เพื่อให้ทำต่อจากเดิมได้ทันทีโดยไม่ต้องไล่ดูใหม่ทั้งหมด

## ภาพรวมระบบ

Multi-tenant chatbot (Facebook + LINE) สำหรับ 3 มหาวิทยาลัย/คลัสเตอร์:
- **Brain 1** (Project_1) = มหาวิทยาลัยพิษณุโลก — เพจ 1
- **Brain 2** (Project_2) = มหาวิทยาลัยชินวัตร — เพจ 2 (ทั่วไป: HR/วิศวะ/ป.โท/ป.บัณฑิต), เพจ 3 (พยาบาล: RN/PN)
- **Brain 3** (Project_3) = ศึกษาต่อจีน — เพจ 4-7 (กว่างโจว/เซี่ยงไฮ้/เวยไห่/ฮาร์บิน)

Middleware: `server.js` (Express) → ตัดสินใจส่งไป **Dify** หรือ **n8n** ตาม `.env`:
```
USE_N8N_BRAIN=false
N8N_BRAIN_CAMPUSES=Project_1,Project_2   <-- ตอนนี้ Brain 1+2 ใช้ n8n จริงแล้ว, Brain 3 ยังอยู่ Dify
```

n8n รันบน Docker (`http://localhost:5678`), workflow ID:
- Brain 1: `SQoyCRRuMOfR7btN`
- Brain 2: `BamyhbNkDkMN4d5C`
- Brain 3: `Az8Uz28lk2JgYrWv` (ยังไม่แตะวันนี้)

Redis (`redis-6380`, port 6380) เก็บ state แยก namespace:
- `conv:<platform>:<userId>` = Dify conversation_id
- `n8n:phitsanulok:<userId>` / `n8n:shinawatra:<userId>` / `n8n:china:<userId>` = state ของแต่ละ Brain ใน n8n
- `bot_control:<campus>` = เปิด/ปิดบอท + ตั้งเวลา
- `applied:<platform>:<userId>` = ลูกค้าเคยสมัครแล้ว (TTL 1 ปี)

## งานที่ทำเสร็จวันนี้

### server.js (ฟีเจอร์ระดับระบบ ใช้กับทุก Brain)
1. **เปิด/ปิดบอท + ตั้งเวลาทำงาน** ต่อแคมปัส
   - `GET/POST /api/admin/bot-control` (Basic Auth: ดู `.env` → `ADMIN_USER`/`ADMIN_PASS`)
   - `enabled:false` = ปิดทันที, `scheduleEnabled:true` = เช็คช่วงเวลา `startTime`-`endTime` (รองรับข้ามเที่ยงคืน)
2. **ตรวจลูกค้าเคยสมัครแล้ว** — แทนที่จะ mark ตอนเห็นลิงก์ฟอร์มในคำตอบ (เคยทำแบบนั้นแล้วพบว่าตัดบทเร็วเกินไป ลูกค้าถามต่อรองไม่ได้) **เปลี่ยนเป็น mark ตอนลูกค้าส่งรูป/ไฟล์ และเช็คว่าห้องแชทนี้เคยได้รับลิงก์ฟอร์มมาก่อนหรือยัง** (`db.hasSentFormLink(sessionId)` ใน `database.js` — เช็คทั้ง `forms.gle` และ `docs.google.com/forms`)
   - `POST /api/admin/clear-applied { platform, userId }` ใช้ล้าง flag ทดสอบ
   - ข้อความตอบ: `"รับทราบค่ะนักศึกษา อาจารย์ดำเนินการติดต่อแอดมินที่ดูแลและตรวจสอบข้อมูลการสมัครให้นะคะ"`
3. **คำสั่งรีเซ็ต** ขยายให้ครอบคลุม typo: `เริ่มใหม่, เริ่มต้นใหม่, ล้างแชท, ล้างแชต, ล้างแชส, reset, restart, clear` — และสำคัญมาก: **เวลา reset ต้องแทนข้อความลูกค้าด้วย "สวัสดีค่ะ" ก่อนส่งต่อให้ AI** (ห้ามส่งคำว่า "เริ่มใหม่" ตรงๆ ให้ AI เพราะ AI จะตีความเป็นคำถามจริงและตอบแปลกๆ)
4. **ActiveUsers.js** แก้ token ให้ใช้ต่อ tenant ถูกต้อง

### Brain 1 (พิษณุโลก) — ผ่าน n8n เต็มรูปแบบ, verified ทุก flow
ไฟล์ build: `build_brain1_update.js` (เครื่องมือ regenerate Extract Info + AI prompt node)
- ดึงสไตล์การตอบจาก log แชทแอดมินจริง: เรียก "นักศึกษา"/"อาจารย์", ห้ามใช้ "ท่าน"/"คุณ"/"ครับ"
- คำทักทายแรกใช้ template ตายตัว (เป๊ะ 100%) แยก 2 บับเบิลด้วย `===`
- ข้อมูลครบ+ไม่รู้สาขา → ถามแค่สาขา ไม่ทักทายซ้ำ
- Deflect คำถามค่าเทอมตอนข้อมูลยังไม่ครบ
- รายละเอียดหลักสูตร (นิติ/บัญชี/รัฐประศาสนศาสตร์) ส่งเป็น 3 บับเบิล: อธิบาย+ส่วนลด → รูปตาราง → ปิดท้ายด้วยคำถามสมัคร+ข้อความกระตุ้นโควต้า (ตรงสไตล์แอดมินจริง)
- ส่วนลด 90,000 บาท ใช้เฉพาะข้าราชการ/รัฐวิสาหกิจจริง (เช็คจาก `job` ที่ลูกค้าระบุ)
- ป.โท/ป.เอก/ป.บัณฑิต มี flow คัดกรองเฉพาะ (ป.โทต้องมีใบประกอบวิชาชีพครู)
- ข้อความส่งฟอร์ม + เลขบัญชีธนาคารกรุงไทย แยกเป็น 2 บับเบิล มีขึ้นบรรทัดใหม่จริง

## ของที่ต้องรู้ก่อนแก้ Brain ใดๆ (n8n)

ขั้นตอนมาตรฐานในการแก้ Brain 1/2 (ดู `build_brain1_update.js` / `build_brain2_update.js` เป็นตัวอย่าง):

```bash
# 1. ดึง workflow ปัจจุบันจาก n8n
curl -s -H "X-N8N-API-KEY: $(grep N8N_API_KEY .env | cut -d= -f2)" \
  "http://localhost:5678/api/v1/workflows/<WORKFLOW_ID>" -o brainX_current.json

# 2. แก้ build_brainX_update.js แล้วรัน
node build_brainX_update.js   # จะสร้าง brainX_updated.json

# 3. ตรวจ syntax ในเครื่องก่อนเสมอ (ห้ามข้าม!)
node -e "... new Function('\$json', 'return ' + expr + ';') ..."

# 4. ส่งเข้า n8n จริง
curl -s -X PUT -H "X-N8N-API-KEY: ..." -H "Content-Type: application/json" \
  --data-binary @brainX_updated.json \
  "http://localhost:5678/api/v1/workflows/<WORKFLOW_ID>"
curl -s -X POST -H "X-N8N-API-KEY: ..." \
  "http://localhost:5678/api/v1/workflows/<WORKFLOW_ID>/activate"

# 5. ทดสอบยิงตรงก่อนปล่อยลูกค้าจริง
curl -s -X POST -H "Content-Type: application/json" \
  -d '{"userId":"diag_test1","message":"..."}' \
  "http://localhost:5678/webhook/<phitsanulok-brain|shinawatra-brain|china-brain>"
```

**กฎเหล็ก**: ห้าม PUT เข้า n8n โดยไม่ตรวจ syntax ในเครื่องก่อน (`new Function`) — เคยทำพังจนลูกค้าจริงโดน error มาแล้ว

**ข้อจำกัดของ gpt-4o-mini ที่เจอบ่อย**: บางครั้งไม่ทำตามคำสั่ง "ตอบเป๊ะๆ" 100% โดยเฉพาะถ้า history มีบทสนทนาคล้ายกันมาก่อน (AI จะเลี่ยงพูดซ้ำเอง) — ถ้าจุดไหนต้องเป๊ะจริงๆ (เลขบัญชี, ลิงก์ฟอร์ม) ให้เขียนคำสั่งเน้นย้ำแบบ "ห้ามแก้ไข ห้ามแต่งเพิ่ม ห้ามตัดคำ" และตรวจผลจริงหลัง deploy เสมอ

## วิธี restart server.js

```bash
# หา process เดิมแล้วฆ่า (PowerShell)
$p = Get-NetTCPConnection -LocalPort 3005 | Where-Object State -eq 'Listen' | Select -Expand OwningProcess
Stop-Process -Id $p -Force

# รันใหม่ (Bash)
node server.js > live_monitor.log 2>&1 &
```

⚠️ ตอนนี้รันแบบ plain `node` ใน background ไม่ใช่ PM2/systemd — ถ้าเครื่อง/เทอร์มินัลปิด เซิร์ฟเวอร์จะหยุดทำงาน ควรพิจารณาตั้ง PM2 ถ้าจะใช้งานจริงต่อเนื่อง (มี `ecosystem.config.js` อยู่แล้วแต่ยังไม่ได้ใช้)
