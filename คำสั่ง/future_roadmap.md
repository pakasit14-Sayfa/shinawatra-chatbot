# แผนการพัฒนาระบบ Chatbot (Backend & Integration)
*เอกสารนี้เก็บบันทึกสิ่งที่ต้องทำในอนาคต หลังจากออกแบบหน้าเว็บ (UI) เสร็จเรียบร้อยแล้ว*

## 1. ดึงรูปโปรไฟล์จริงของลูกค้า (Profile Picture Sync)
*   **เป้าหมาย:** แสดงรูปโปรไฟล์ที่แท้จริงของลูกค้าจาก Facebook และ LINE แทนการใช้รูปอวตารจำลอง (Mock Avatar)
*   **สิ่งที่ต้องทำ:** 
    *   แก้ไข API `fetchChatData` ฝั่งหลังบ้านให้รองรับการส่ง URL รูปภาพกลับมาที่หน้าบ้าน
    *   อัปเดตโมเดลหรือโครงสร้างฐานข้อมูล (ถ้าจำเป็น) ให้เก็บ `profile_pic_url` ของ `User`

## 2. เซ็ตอัประบบฐานข้อมูลจำลอง (Local Database Setup)
*   **เป้าหมาย:** ให้สามารถรันระบบและทดสอบแชทเข้า-ออกแบบสมจริงบนเครื่องคอมพิวเตอร์ส่วนตัว (Local) ได้
*   **สิ่งที่ต้องทำ:** 
    *   เชื่อมต่อ `DATABASE_URL` (MySQL/PostgreSQL) สำหรับ Local
    *   รันคำสั่ง `npx prisma db push` เพื่อสร้างตารางให้ตรงกับ Schema ปัจจุบัน
    *   สร้างข้อมูลจำลอง (Seed Data) สำหรับการทดสอบ

## 3. เชื่อมต่อระบบ AI Assistant (Dify Integration)
*   **เป้าหมาย:** ให้ AI (ผ่านแพลตฟอร์ม Dify) สามารถตอบกลับลูกค้าได้อัตโนมัติก่อนที่จะส่งต่อให้แอดมิน
*   **สิ่งที่ต้องทำ:** 
    *   สร้าง API Endpoint ใหม่สำหรับรับส่งข้อความกับ Dify
    *   ใช้ `dify_api_key` ที่เก็บไว้ในตาราง `AiAgent` เพื่อยืนยันตัวตน
    *   สร้างระบบ "Handoff" (สลับการสนทนาระหว่าง AI กับ แอดมินที่เป็นมนุษย์)

## 4. นำโค้ดขึ้นระบบใช้งานจริง (Deployment / Push to Production)
*   **เป้าหมาย:** อัปเดตโค้ดที่พัฒนาเสร็จแล้วไปไว้บนเครื่อง Server หลัก
*   **สิ่งที่ต้องทำ:** 
    *   ใช้ Git ในการ Commit และ Push โค้ด
    *   รัน `npm run build` และ Restart Service ฝั่งเครื่องหลัก
