# 📝 บันทึกกระบวนการสร้าง Dify Workflow (Study in China)
**วันและเวลาที่บันทึก:** 20 มิถุนายน 2026 เวลา 08:49 น.

## 🧠 สิ่งที่คิด (Thought Process)
เมื่อได้รับมอบหมายให้สร้างระบบแชทบอท Dify สำหรับการแนะนำเรียนต่อในประเทศจีน (4 เมือง: กว่างโจว, เซี่ยงไฮ้, เวยไห่, ฮาร์บิน) สิ่งที่ผมวิเคราะห์และคิดเพื่อวางโครงสร้างระบบคือ:
1. **ความง่ายในการใช้งาน (User Experience):** ลูกค้าควรได้รับคำถามที่เป็นขั้นเป็นตอน ไม่ถามรวบรัดเกินไป และ AI ต้องส่งต่อข้อมูลได้โดยที่ลูกค้าไม่ต้องพิมพ์ข้อมูลซ้ำ
2. **สถาปัตยกรรมที่ยืดหยุ่น (Scalable Architecture):** แทนที่จะใช้ AI เพียงตัวเดียวตอบทุกเรื่อง การแยก AI ผู้เชี่ยวชาญออกเป็น 4 ตัว (ประจำ 4 เมือง) จะทำให้เขียน Prompt ได้เฉพาะเจาะจง ควบคุมคำตอบได้ง่ายกว่า และแม่นยำกว่า ลดอาการเดามั่ว (Hallucination)
3. **การส่งต่อข้อมูลแบบลับๆ (Tag Routing):** ใช้เทคนิค "Router LLM" เพื่อซักถามเบื้องต้น และแปะ Tag (เช่น `[สถานะ:กว่างโจว]`) เพื่อให้ระบบหลังบ้านใช้แยกทางรถไฟ (IF/ELSE) ได้อย่างแม่นยำ

## ⚙️ กระบวนการทำงาน (Working Process)
1. **ศึกษาข้อมูล (Data Analysis):** อ่านเอกสารเงื่อนไขการรับสมัคร, ค่าใช้จ่าย, วีซ่า, และหอพักของทั้ง 4 เมือง (ไฟล์ 01-04) รวมถึงแพตเทิร์นการตอบแชทที่คุณสายฟ้าให้มา
2. **คัดลอกและดัดแปลงแม่แบบ (Templating):** นำไฟล์ YAML ของมหาวิทยาลัยชินวัตรที่เสถียรอยู่แล้ว มาดัดแปลงโครงสร้าง แทนที่จะลากเส้นใหม่ตั้งแต่ต้น เพื่อป้องกัน Error ของ Node ID
3. **เขียน Prompt ใหม่ (Prompt Engineering):**
   - **Router:** สั่งให้ถามข้อมูลพื้นฐาน (อายุ, พื้นฐานจีน, ประวัติ) และเสนอ 4 เมือง ก่อนจะคาย Tag
   - **LLMs ประจำเมือง:** สั่งให้ AI แต่ละตัวจำข้อมูลค่าใช้จ่ายและจุดเด่นของมหาวิทยาลัยในเมืองนั้นๆ พร้อมทั้งเสนอทางเลือกระยะเวลาเรียน (1 เทอม / 1 ปี)
4. **ทำความสะอาดโค้ด (Code Cleanup):** เขียนสคริปต์ Python ลบเส้นทางและ Node ของ "ผู้ช่วยพยาบาล" ที่ติดมาจากแม่แบบเดิมทิ้ง เพื่อให้เหลือเฉพาะ 4 เมืองที่ใช้งานจริง
