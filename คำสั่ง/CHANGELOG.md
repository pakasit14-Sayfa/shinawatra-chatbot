## [2026-06-22] - เพิ่มระบบความปลอดภัยและการจำลูกค้า

### Added (เพิ่ม)
- **Redis Integration:** ระบบการเชื่อมต่อกับ Redis เพื่อเซฟและดึง `conversation_id` ของลูกค้า ป้องกันไม่ให้ AI เริ่มคุยใหม่ทุกครั้งที่ Server รีสตาร์ทหรือมีการ Deploy โค้ดใหม่
- **LINE Signature Verification:** ตรวจสอบความถูกต้องของ Webhook ที่ส่งเข้ามาทางเส้นทาง `/webhook/line/:campus` ด้วยการคำนวณ Hash ของ Request Body เทียบกับ `x-line-signature` ช่วยบล็อก Request ปลอมที่ไม่ได้มาจากเซิร์ฟเวอร์ของ LINE จริงๆ
- ตัวแปรใน `.env.example` และ `.env`: เพิ่ม `LINE_SECRET_1`, `LINE_SECRET_2`, `LINE_SECRET_3` สำหรับตั้งค่า Channel Secret เพื่อตรวจสอบลายเซ็น

### Changed (แก้ไข)
- **`config.js`:** เพิ่มฟิลด์ `lineChannelSecret` ไปใน Config ของ LINE แต่ละเพจ (line_1, line_2, line_3) เพื่อรองรับฟีเจอร์ Multi-Tenant สำหรับการตรวจสอบลายเซ็น
- **`server.js`:** 
  - ย้ายการส่งสถานะ `res.sendStatus(200)` ไปไว้ *หลัง* ขั้นตอนการตรวจสอบลายเซ็น เพื่อให้สามารถบล็อก Request ได้จริงด้วยสถานะ `403 Forbidden`
  - ปรับการทำงานของฟังก์ชัน `sendToDify` ให้สามารถอ่านและเขียนค่าลง Redis ผ่าน `await` แบบ Asynchronous

### Fixed (แก้บั๊ก)
- **`server.js`:** แก้บั๊กการประกาศตัวแปรในส่วนรับข้อความ LINE ที่ใช้คำสั่ง `const text = ...` แล้วมีการแก้ค่า (Reassign) ทีหลังด้วยฟังก์ชัน `sanitizeInput()` ซึ่งก่อให้เกิด `TypeError: Assignment to constant variable` ได้เปลี่ยนเป็น `let text = ...` ทำให้โค้ดรันได้ปกติ

### Brain 2 (ชินวัตร) — รูปแบบการตอบ
- ปรับ ป.โท/ป.บัณฑิต/วิศวกรรม/HR จากพารากราฟยาวเป็น bullet list + อีโมจิหมวด (📍🗓💰📄) ตามสไตล์แอดมินจริงที่ดึงมาจาก CSV log จริงและไฟล์ `รายละเอียดหลักสูตรทุกสาขา.docx` ที่แอดมินเขียนไว้
- เพิ่มกฎกลางให้ AI จัด bullet list เองทุกครั้งที่ต้องเสนอตัวเลือกหลายอย่าง (ไม่ต้องเขียน template ตายตัวทุกจุด)
- เพิ่ม whitelist คำถามง่ายๆที่ตอบตรงได้ทันที (ไม่ deflect): "ยากไหม", "ออนไลน์ไหม", "คุรุสภา/กคศ. รับรองไหม" (พร้อมลิงก์ตรวจสอบจริง), "มหาวิทยาลัยอยู่ที่ไหน", "อยู่ต่างจังหวัดเรียนได้ไหม/ออนไซต์บ่อยแค่ไหน" — เฉพาะเพจทั่วไป ไม่แตะฝั่งพยาบาล
- เพิ่ม "การจัดการ" เป็นคำที่จับเข้าสาขา HR (คณะการจัดการ)

### Brain 2 — แก้ตรงกับเอกสารคำสั่งสอนจริงของแอดมิน (3 จุด)
- **ป.บัณฑิต:** ขยายเงื่อนไขรับสมัครจาก "เฉพาะครู สพฐ." เป็น "สพฐ. หรือเอกชนที่มีการเรียนการสอนปกติ" (เคยปฏิเสธลูกค้าที่ควรผ่านเกณฑ์)
- **วิศวกรรมความปลอดภัย:** แก้ระยะเวลาวุฒิ ม.6 จาก "3 ปี" (ผิด) เป็น "4 ปี" (ตรงเอกสารจริง)
- **HR (การจัดการทรัพยากรมนุษย์):** สร้างระบบคำนวณระยะเวลาเทียบโอนใหม่ทั้งหมดตามตารางจริง — แยกตามสายที่จบ (ปวส./ป.ตรี: บริหารธุรกิจ=1.5ปี, สังคม=2ปี, วิทย์=2.5ปี) และปีประสบการณ์ทำงาน (กศน./ม.6/ปวช.: 0ปี=3.5ปี, 1-11ปี=3ปี, 12ปีขึ้นไป=2.5ปี)
- **ป.โท:** ตัดเงื่อนไขขอวุฒิการศึกษาออก เหลือถามแค่ "มีใบประกอบวิชาชีพครูหรือไม่" ตามที่ทีมยืนยัน

### Brain 2 — บั๊กจริงที่เจอจากบทสนทนาลูกค้าจริงใน Redis + CSV
- **ลูกค้าตอบ "ครับ"/"ค่ะ" เปล่าๆ** หลังเห็นรายละเอียดคอร์สแล้ว ไม่ถูกนับเป็นการยืนยันสมัคร (ไม่ส่งฟอร์ม+QR ให้) — กระทบยอดสมัครจริง แก้แล้ว
- **AI มั่วตอบว่า "ข้อมูลครบแล้ว สมัครได้เลย"** ทั้งที่ยังไม่ได้ถามวุฒิ (ป.โท) — แปลงขั้นถามข้อมูลที่ขาดเป็น forcedAnswer ตายตัว ไม่ผ่าน AI อีก
- **"ยังครับ" เปล่าๆ ไม่ถูกตีความเป็น "ไม่มี"** (เฉพาะ "ไม่มี/ยังไม่มี" ที่ regex จับได้) ทำให้ถามซ้ำไม่จบ — เพิ่มการจับคำตอบสั้นแบบนี้แล้ว
- **เลขเกรดแบบเต็มจำนวนไม่มีจุด (เช่น "3")** ไม่ถูกจับเป็นเกรด (regex เดิมจับแค่ทศนิยม) ทำให้คนพิมพ์สั้นๆติดวนไม่จบ — แก้แล้ว
- **Auto-pick หลักสูตร/ทางเลือกโดยไม่ถามยืนยัน** เมื่อลูกค้าพูดถึง 2 ตัวเลือกพร้อมกันในคำถามเปรียบเทียบ (เช่น "เครื่องกลกับความปลอดภัยต่างกันยังไง" หรือ "รัฐกับเอกชนต่างกันยังไง") — แก้ที่ Engineering, HR, และ (ได้รับอนุญาตเฉพาะจุดนี้) PN's pnTrack
- **AI ตอบคำถามแทรกนอกขอบเขตเอง** (เช่น ถามรายการเอกสารที่ต้องใช้) ทั้งที่ควร deflect ไปแอดมิน — แปลง offTopic deflect เป็น forcedAnswer ตายตัว ไม่ผ่าน AI อีกต่อไป
- เพิ่มรูปตารางค่าเทอมวิศวกรรม (6 รูป: เครื่องกล/ความปลอดภัย/2ปริญญา × เทียบโอน/ไม่เทียบโอน) และป.บัณฑิต (1 รูป) แนบอัตโนมัติตามวุฒิ/หลักสูตรที่ลูกค้าเลือก
